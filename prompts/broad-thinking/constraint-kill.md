# Constraint Kill

## Purpose
Remove all constraints to discover the ideal solution, then work backwards to reality.

## Prompt Template

```
Let's think about [PROBLEM] with zero constraints.

**Phase 1 — The Ideal World**
Assume:
- Unlimited budget and time
- Any technology exists and works perfectly
- You can change any API, language, framework, or standard
- Users will adopt whatever you build
- There are no legacy systems or backwards compatibility concerns

What does the perfect solution look like? Describe it in detail.

**Phase 2 — The Reality Filter**
Now bring back reality, one constraint at a time:
1. What parts of the ideal solution are actually achievable today?
2. What parts could be achievable in 6 months?
3. What parts require a breakthrough that doesn't exist yet?
4. What's the 80/20 version — 80% of the ideal value with 20% of the effort?

**Phase 3 — The Stepping Stones**
Design a path from today to the ideal:
- What can you ship this week that moves toward the ideal?
- What's the minimum viable version that proves the concept?
- What's the first constraint you'd want to remove?
```

## When to Use
- When over-constrained thinking is blocking progress
- Product strategy and roadmap planning
- When the current solution feels like a local optimum
