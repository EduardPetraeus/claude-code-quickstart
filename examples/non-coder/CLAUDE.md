# CLAUDE.md — Writing & Analysis Project

## Project Context

- **Repo:** policy-analysis
- **Purpose:** Research and writing workspace for policy documents, reports, and analysis
- **Stack:** Markdown, Python (data analysis), Excel/CSV
- **Status:** Ongoing

## Language

- All documents and analysis in English
- Respond in the language I write to you in

## My Role

I am a policy analyst / economist / HR specialist. I am NOT a programmer.
When I need code, write it for me with clear comments explaining what it does.
Prefer simple scripts over complex solutions.

## Writing Style

- Professional but accessible — avoid jargon unless the audience expects it
- Active voice over passive voice
- Short paragraphs (3-5 sentences max)
- Use headers, bullet points, and tables for structure
- Always cite sources with links where available
- Executive summary first, details later

## Document Structure

- Reports go in `reports/`
- Data files go in `data/`
- Analysis scripts go in `scripts/`
- Drafts go in `drafts/`

## When I Ask for Data Analysis

- Use Python with pandas for data work
- Create simple, readable charts with matplotlib
- Save outputs to `outputs/`
- Explain every step in comments — I need to understand what the code does
- Keep scripts short and focused — one script per analysis

## When I Ask for Writing Help

- Match the tone of the document I'm working on
- Suggest structure before writing content
- Flag claims that need citations
- Keep original meaning when editing — don't add your own opinions

## Session Protocol

- Start by reading what I'm working on (I'll point you to the file)
- Ask clarifying questions before making changes
- Show me the plan before executing
- Summarize changes at the end

## Boundaries

- Never fabricate data, statistics, or citations
- Never commit files without asking me first
- If you're unsure about something, say so — don't guess
