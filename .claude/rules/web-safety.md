# Web Safety

WebFetch and WebSearch are auto-allowed. Still ASK before proceeding if:

- The URL contains tokens, API keys, or credentials in query params
- The site requires login or sending credentials
- The domain looks like phishing or is completely unknown/suspicious
- The request would POST data (not just read)
- The URL is an `.onion` address, raw IP, or obfuscated

Everything else (docs, GitHub, Stack Overflow, blogs, news, package registries, etc.) — just do it.
