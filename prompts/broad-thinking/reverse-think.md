# Reverse Think

## Purpose
Invert the problem to discover hidden risks, blind spots, and unconventional solutions.

## Prompt Template

```
Let's think about [PROBLEM/PROJECT] in reverse.

**1. Guaranteed Failure Plan**
How would we guarantee this project fails? List the top 5 things we could do to ensure failure:
- (Think: bad decisions, wrong priorities, ignored risks)

**2. The Anti-User**
If someone wanted to have the worst possible experience with our product, what would they do? What does that reveal about our design?

**3. The Competitor's Playbook**
If a well-funded competitor wanted to make our product irrelevant, what would they build? What does that tell us about our moat (or lack thereof)?

**4. The Regret Test**
Fast forward 1 year. This project failed. Write the post-mortem:
- What went wrong?
- What warning signs did we ignore?
- What should we have done differently?

**5. The Inversion**
Now flip every failure point into an action item:
- Each guaranteed-failure move → what's the opposite action?
- Each anti-user pain point → what's the fix?
- Each competitor threat → what's our defensive move?
```

## When to Use
- Pre-mortem before starting a project
- Risk assessment for major decisions
- When optimism bias is clouding judgment
- Before committing resources to a direction
